# まいカゴ 成長・収益化改善設計書

**作成日**: 2026-03-10
**ステータス**: 承認済み

## 背景・課題

まいカゴはインストール数がほぼゼロ、課金ユーザーもいない状況。
技術的には優れている（OCR + AI + リアルタイム同期）が、以下の構造的問題がある：

| 課題 | 詳細 |
|------|------|
| 初回体験の壁 | Googleログイン必須で離脱率が高い |
| プレミアム特典の魅力不足 | 広告非表示・テーマ・フォントのみ。実用的価値がない |
| 攻撃的な広告 | 3操作ごとのインタースティシャル広告がUXを破壊 |
| ストア訴求のミスマッチ | 「値札読み取り」がニッチすぎて検索にひっかからない |
| ターゲットの曖昧さ | 誰の何を解決するアプリか不明確 |

## ターゲットユーザー

**予算内で計画的に買い物を完了させたい節約志向の人**

- 買うものを事前に決めて、それだけを買いたい
- 合計金額が予算をオーバーしていないか常に把握したい
- 現在の代替手段：メモアプリ＋電卓

## 価値提案

**「メモと電卓はもういらない。買い物リスト＋自動計算＋予算管理、これ1つ。」**

## 設計

### 1. 初回体験の再設計（オンボーディング）

#### ウェルカム画面（チュートリアル兼価値訴求、3枚スライド）
1. 「買い物リスト＋電卓、これ1つで」— 価値を3秒で伝える
2. 「予算を設定して、オーバーしない買い物を」— 予算機能の訴求
3. 「値札を撮るだけで自動入力」— OCRのデモ

#### ゲストモード
- 「まずは使ってみる」ボタンでログイン不要で即使える
- ローカルストレージでリスト管理
- ショップ1つ、アイテム追加・合計計算・予算設定は使える
- OCRは1回お試し可能
- 価値を実感したタイミングでログイン促進（「データをクラウドに保存しませんか？」）
- ゲストモードのデータはログイン後にFirestoreへマイグレーション
- ウェルカム画面は初回のみ表示（SharedPreferencesでフラグ管理）

### 2. プレミアム特典の再設計

#### 無料版
- リスト管理（アイテム追加・編集・削除・チェック）— 無制限
- 合計金額の自動計算 — 無制限
- 予算設定と残額表示 — 無制限
- 並び替え（手動・価格順・名前順等）— 無制限
- クラウド同期（ログイン時）— 無制限
- **ショップ（タブ）：2つまで**
- **OCR（値札撮影）：月5回まで**
- テーマ：4種類（pink, dark, light, lemon）
- レシピ解析 — 利用不可
- 共有グループ（ファミリー機能）— 利用不可
- バナー広告あり（画面下部のみ）

#### プレミアム（買い切り ¥480）
- 無料版の全機能
- ショップ（タブ）：無制限
- OCR（値札撮影）：無制限
- レシピ解析
- 共有グループ（ファミリー機能）
- 全テーマ・全フォント
- 広告完全非表示

#### 制限設計の意図
| 制限 | 狙い |
|------|------|
| ショップ2つまで | 複数店を使い分ける節約上手な人ほど課金動機が生まれる |
| OCR月5回 | 便利さを体験させた上で制限。手入力に戻る不便さが課金を後押し |
| レシピ解析をプレミアム限定 | APIコスト（OpenAI）がかかる機能をプレミアムに集約 |
| 共有をプレミアム限定 | 家族利用=継続利用が見込める。最も課金意欲の高いセグメント |

#### 価格変更の理由
- ¥280 → ¥480：特典の実用的価値が大幅に上がるため
- 買い切りで¥480は「コーヒー1杯分」で訴求しやすい
- 7日間無料体験はそのまま維持

### 3. 広告戦略の変更

| 変更 | 理由 |
|------|------|
| バナー広告のみ残す（画面下部） | 許容されやすく、UXを守れる |
| インタースティシャル広告を廃止 | 3操作ごとの全画面広告はアンインストールの主因 |
| アプリオープン広告を廃止 | 起動時の全画面広告はユーザー体験を損なう |

### 4. ストア訴求の再設計（ASO）

#### アプリ名
```
現在: まいカゴ – 値札読み取りで買い物の合計金額が一瞬でわかる
変更: まいカゴ – 買い物リスト＆電卓。予算オーバーしない
```

#### ストア短い説明（80文字）
```
メモと電卓はもういらない。買い物リスト＋自動計算＋予算管理、これ1つ。
```

#### スクリーンショット構成（5枚）
1. 「メモ＋電卓、これ1つで」— リスト画面＋合計表示のメイン画面
2. 「予算を決めて、残額を見ながら買い物」— 予算設定＋残額バー
3. 「値札を撮るだけで自動入力」— カメラ→OCR→リストに追加の流れ
4. 「お店ごとにリストを分けて管理」— タブ表示の切り替え
5. 「家族で共有、リアルタイム同期」— 共有機能のイメージ

#### ASOキーワード
買い物リスト / 節約 / 予算管理 / 買い物メモ / 家計 / 買い物計算 / 買いすぎ防止 / 食費節約 / お買い物 / ショッピングリスト

### 5. マーケティング施策（月数千円予算内）

| 施策 | コスト | 期待効果 |
|------|--------|----------|
| Google UAC（アプリ広告） | 月2,000〜3,000円 | CPI 50〜100円想定で月20〜60インストール |
| スクリーンショット刷新 | 無料（自作） | CVR改善。最もコスパの高い施策 |
| X（Twitter）での発信 | 無料 | 「節約術」系投稿にアプリを絡める |

## 実行ロードマップ

### Phase 1：プロダクト改善（Week 1-6）

**Week 1-2: 初回体験の改善**
- ゲストモード実装（ローカルストレージでリスト管理）
- ウェルカム画面（チュートリアル兼価値訴求）
- ゲスト→ログイン時のデータマイグレーション

**Week 3-4: プレミアム再設計**
- OCR月5回制限の実装（カウンター + UI表示）
- ショップ2つ制限の実装
- レシピ解析をプレミアム限定に変更
- 共有グループをプレミアム限定に変更
- プレミアム紹介画面の刷新（新特典を訴求）
- 価格を¥480に変更

**Week 5-6: 広告戦略の変更**
- インタースティシャル広告の廃止
- アプリオープン広告の廃止
- バナー広告のみ残す

### Phase 2：ストア最適化（Phase 1と並行〜完了後）

**Week 3〜: ストア掲載の改善（Phase 1と並行で準備可能）**
- アプリ名変更
- ストア説明文の書き直し
- スクリーンショット5枚作成
- ASOキーワード設定
- リリース（Phase 1の実装と同時にストア反映）

**Week 7〜: マーケティング開始**
- Google UAC 月2,000〜3,000円で開始
- X（Twitter）での発信開始
- インストール数・CVRを週次でモニタリング

### Phase 3：データドリブン改善（3ヶ月目〜）
- CVRが低い → スクリーンショット・説明文をA/Bテスト
- インストール後の離脱が多い → ウェルカム画面・初回体験を調整
- 課金率が低い → 制限の度合いや価格を調整
- 課金率が良い → 機能拡張の検討開始

## 成功指標

|------|------|-------------|-------------|
| 月間インストール | ほぼ0 | 50〜100 | 200〜500 |
| 7日継続率 | 不明 | 30%以上 | 40%以上 |
| プレミアム課金率 | 0% | 3〜5% | 5〜10% |
| 月間収益 | ¥0 | ¥1,000〜2,000 | ¥5,000〜10,000 |

## リスクと対策

| リスク | 対策 |
|--------|------|
| OCR制限でユーザーが離脱 | 月5回で試し、離脱率が高ければ月8回に緩和 |
| ¥480が高いと感じられる | ¥280に戻すオプションを残す。セール施策も検討 |
| ゲストモードの実装が複雑 | ローカルのみのシンプルな実装に留める。同期は後回しでもOK |
| API無料枠を超過 | OCR制限がそのまま防壁になる。プレミアムユーザー増加でコスト回収 |
