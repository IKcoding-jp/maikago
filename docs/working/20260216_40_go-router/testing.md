# テスト計画: ナビゲーション管理の導入（go_router）

## テスト方針

本Issueはリファクタリング（内部構造の変更）であり、外部動作は変わらないため、既存テストが通ることが最低条件。

## 自動テスト

### 既存テスト
- `flutter test` で全既存テストが通ること
- 特にナビゲーション関連のテストがある場合は重点確認

### 静的解析
- `flutter analyze` でエラー・警告なし

## 手動確認項目

### 認証フロー
- [ ] 未ログイン状態でアプリ起動 → ログイン画面が表示される
- [ ] ログイン後 → メイン画面に遷移する
- [ ] ログアウト → ログイン画面に戻る

### ドロワーメニュー遷移
- [ ] ドロワーから各画面（About, 使い方, 電卓, サブスク, フィードバック, 更新履歴, 設定）に遷移できる
- [ ] 各画面から戻るボタンでメイン画面に戻れる

### 設定画面遷移
- [ ] 設定画面から各サブ画面に遷移できる
- [ ] サブ画面から設定画面に戻れる

### カメラフロー
- [ ] カメラ画面を開ける
- [ ] OCR結果確認画面で結果を選択してメイン画面に戻れる
- [ ] レシピ確認画面で選択してメイン画面に戻れる

### ダイアログ（影響がないこと）
- [ ] 予算設定ダイアログが正常に開閉する
- [ ] アイテム編集ダイアログが正常に動作する
- [ ] その他ダイアログに影響がない

### Web対応
- [ ] Web版で横幅800px制限が維持されている
- [ ] ブラウザのURL欄にルートパスが反映される
- [ ] ブラウザの戻る/進むボタンが動作する

## リスク
- 戻り値付き遷移（pop with result）の動作が変わる可能性
- ドロワーの閉じるタイミングが変わる可能性
- Web版のURL変更によるFirebase Hosting設定の影響
