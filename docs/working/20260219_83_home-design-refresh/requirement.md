# 要件定義

**Issue**: #83
**作成日**: 2026-02-19
**ラベル**: enhancement

## ユーザーストーリー
ユーザー「ホーム画面が安っぽく見えるので、もっと洗練されたデザインにしてほしい」
アプリ「3パターンのデザインモックを提示し、ユーザーが気に入ったパターンで画面を刷新する」

## 要件一覧
### 必須要件
- [ ] 3パターンのデザインモック（HTML）を作成する
- [ ] 各パターンは明確にコンセプトが異なること
- [ ] 現在の機能（タブ切替、2列リスト、予算表示、アクションボタン）を維持すること
- [ ] 既存の16テーマシステムとの互換性を維持すること
- [ ] レスポンシブ対応（モバイル〜Web 800px制限）を維持すること
- [ ] 採用パターンをFlutterで実装すること

### オプション要件
- [ ] アニメーション・マイクロインタラクションの追加
- [ ] ダークモード対応の改善
- [ ] アイコン・イラストの刷新

## 現在のUI構成

### AppBar（MainAppBar）
- 横スクロール可能なタブ（ショップ名）
- タブ追加ボタン（アイコン + テキスト）

### Body（ItemListSection）
- 2列分割: 未購入（左）| 購入済み（右）
- 各列: ヘッダー（タイトル + ソート/削除ボタン）+ ReorderableListView
- リストアイテム: Card（アイテム名、価格、数量、割引表示）

### BottomSummaryWidget
- アクションボタン行: 予算変更 / カメラ / レシピ / リスト追加
- サマリーエリア: 残り予算 | 合計金額

### Drawer（MainDrawer）
- アプリアイコン + タイトル
- メニュー項目（About, 使い方, 電卓, プレミアム等）

## 受け入れ基準
- [ ] 3パターンのHTMLモックが作成されている
- [ ] ユーザーが1パターンを採用決定した
- [ ] 採用パターンがFlutterで実装されている
- [ ] 既存の全機能が正常に動作する
- [ ] 既存テーマ（16種類）で正しく表示される
- [ ] flutter analyze でエラーなし
- [ ] flutter test が全パス
