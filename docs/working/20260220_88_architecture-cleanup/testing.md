**作成日**: 2026-02-20

## テスト方針

本Issueはリファクタリングのため、**既存の動作を変更しないこと**が最重要。
新規機能テストではなく、既存テストの通過を主な検証基準とする。

---

## 自動テスト

### 1. 既存テスト通過確認

```bash
flutter test
```

- 全既存テストが通過すること
- リファクタリングにより壊れたテストは修正

### 2. 静的分析

```bash
flutter analyze
```

- エラー・警告ゼロ
- 未使用インポートの解消
