# 要件定義

**Issue**: #131
**作成日**: 2026-03-13
**ラベル**: bug

## ユーザーストーリー

ユーザー「共有タブを設定してアプリを再起動したら、共有してるタブだけが倍になってた」
アプリ「共有状態に関わらず、起動時にタブが正確に表示される」

## 要件一覧

### 必須要件

- [ ] ショップ（タブ）の重複除去メカニズムを実装する
- [ ] `updateSharedGroup()` で全タブ間のクロスリファレンスを正しく更新する
- [ ] リアルタイム同期後にショップの重複が除去されること
- [ ] `loadData()` の二重呼び出しを防止する

### オプション要件

- [ ] `getShops()` Stream版にも重複除去を追加

## 受け入れ基準

- [ ] 共有タブを設定した状態でアプリを再起動しても、タブ数が変わらないこと
- [ ] 既存の共有グループに新規タブを追加した場合、全タブのクロスリファレンスが正しく更新されること
- [ ] `flutter analyze` がエラーなしで通ること
- [ ] 既存テストが全て通ること
