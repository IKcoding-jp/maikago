# 修正内容の確認：OCR結果確認画面のレイアウト最終調整

ユーザー要望「既存商品 ↓ 読み取り商品 ↓ 上書き更新」の順序に基づき、アクションスイッチの位置を調整しました。

## 実施した変更

### UIレイアウトの最終形
1. **既存商品 (Existing)** - 緑色の枠で表示（最上部）
2. **↓ (矢印)** - データの流れを表現
3. **読み取り商品 (New)** - 編集可能なフィールド（中間）
4. **アクション決定スイッチ** - 「上書き更新 / 別に追加」（最下部）

これにより、
「既存の商品はこれです」
↓
「新しい入力値はこれです」
↓
「で、これで上書きしますか？」
という、ユーザーの思考順序に完全に一致したUIとなりました。

## 検証プラン

### 動作確認手順
1. **フローの確認**:
   - 上から下へ視線を移動させたとき、情報の流れが自然であることを確認。
   - 最後にスイッチで「決定」する操作感になっていることを確認。
