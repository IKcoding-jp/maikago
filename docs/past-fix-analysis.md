## カテゴリ別分析

### 1. テーマ・色の不整合（防止可能度: 高）

**該当コミット:**
- `b57e9c71` fix(theme): ログイン画面をテーマカラーに対応
- `c83eafd3` fix(theme): ダークテーマの文字色・背景色を修正し、全テーマの表示を改善
- `5e99e028` style(#117): 合計金額テキストの配色を改善
- `dec24afe` refactor(#35,#46): テーマ色分岐・SnackBar重複解消、Color/fontSize定数集約
- `83429e6e` refactor(ui): コード品質改善 — 色・フォーマッター・SnackBar・テーマ統一

**根本原因:**
- `Colors.red`, `Colors.grey[800]`, `Colors.black87` 等のハードコードが散在
- ダークモード分岐（`isDark ? X : Y`）がファイルごとに個別実装
- テーマカラーの使い方（`colorScheme.primary`, `colorScheme.error` 等）のルールなし

**CLAUDE.mdで防げたルール:**
```
## 色の使用ルール
- ハードコード色（Colors.red, Colors.grey等）は原則禁止。colorScheme経由で参照すること
- ダークモード分岐は不要。テーマ側で吸収する設計にすること
- 新画面追加時は全テーマ（14種）× ダークモードでの見た目を確認すること
```

- `011415c8` refactor: 自前コーチマーク実装をtutorial_coach_markパッケージに置換

**根本原因:**

- 共通ユーティリティの作成ルールがなかった（SnackBar、フォーマッター等の重複）

**CLAUDE.mdで防げたルール:**
```

- 1ファイル500行を超えたら責務分割を検討すること
- 同じロジックが2箇所以上で使われたら共通ユーティリティに抽出すること
- 新規ユーティリティ作成時は lib/utils/ に配置し、既存ユーティリティと重複しないか確認すること
```

**CLAUDE.mdで防げたルール:**

- リアルタイム同期のStream購読中はバッチ更新でUI更新を抑制すること
- context.pop() 後のawait処理ではmountedチェック必須
